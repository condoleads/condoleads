WHY THIS SLIPPED PAST EARLIER CHUNKS
═══════════════════════════════════════════════════════════════════════════

Chunk 4 rewrote tax-match as SOLD-comp matching using the shared seller-
estimator query helper. The seller side never renders tiles with photos —
the estimator pipeline reads tax/price/beds/dom and computes a number, not
a tile. So the SELECT had no reason to fetch media. The buyer reuses the
same query but THEN renders tiles, which is the new requirement that wasn't
flowed into the query.

Honest no-media was a plausible hypothesis (SOLD listings sometimes lose
media at close). The DB ruled it out: 99.9% coverage on Closed Whitby.
