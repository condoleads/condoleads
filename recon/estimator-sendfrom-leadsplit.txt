    when the rewrite is being performed for a comprehensive-site
    agent — which is NOT the WALLiam hero path. So neither branch
    sets x-tenant-id on the server-action request.

ROOT CAUSE — final
═══════════════════════════════════════════════════════════════════════════

  middleware.ts:108-115 injects x-tenant-id ONLY when pathname
  starts with `/api`. Server actions (submitLeadFromForm) target the
  page route (e.g. /[slug] or /property/[id]/...), not /api/*. The
  estimator's submit handler reads `headers().get('x-tenant-id')` at
  app/actions/submitLeadFromForm.ts:51, gets null, returns
  `{success:false}` before reaching getOrCreateLead. No lead row
  inserted; no email send attempted.

  EVIDENCE:
    (a) Probe 2: last WALLiam lead from `source='estimator'` is
        2026-06-08 (~9 days ago). Charlie path (an /api/* route)
        created leads as recently as 2026-06-16 19:25. Same tenant,
        same agent, same DB — the differentiator is the route type.
    (b) Probe 3: middleware.ts:109 makes this scope-gating
        explicit.
    (c) Probe 1: eliminates the send_from / TenantEmailNotConfigured
        red herring. The email-config path is healthy IF
        submitLeadFromForm succeeds — but it doesn't.

  ELIMINATED (not the cause):
    • agentId silent-skip gate           (RECON 2 already eliminated)
    • FK reject on lead insert           (RECON 2 already eliminated)
    • Missing/invalid Resend key         (RECON 2 already eliminated)
    • send_from = NULL                   (Probe 1 above eliminates)
    • Disabled tenant                    (RECON 2 already eliminated)

  FIX SHAPE (flag only — NO code edits this turn):
    Extend middleware.ts:109 to also inject x-tenant-id for server-
    action POSTs to page routes. Two viable shapes:
      (i)  Detect server-action requests by header (Next.js sets
           `Next-Action` on these) and inject x-tenant-id on those
           too. Tight scope, no over-injection.
      (ii) Always inject x-tenant-id when the request resolves a
           tenant from host. Broader scope; the header becomes
           reliable everywhere. Slightly more attack surface to
           audit for code that conditionally trusts the header.
    Either preserves the existing /api/* path verbatim. Charlie /
    walliam/contact / walliam/estimator/vip-questionnaire paths
    keep working unchanged.
